# API Platform — Key Features

## Core Concept

Semantic documentation layer for AI/MCP. Transforms raw API specs into LLM-optimized content with natural language tool descriptions, MCP config generation, and semantic search.

---

## 1. Project & API Management

- Create, view, and delete projects from a central dashboard
- Each project holds multiple API entries with full metadata
- Filter APIs by status (Stable, Beta, Deprecated, Internal) and tags
- Group APIs by functional group
- Per-project API counts tracked in sidebar

## 2. AI-Powered Documentation Generation

- Generate `tool_description` (natural language intent for LLM agents) via Claude API
- Auto-generate `mcp_config` (complete Model Context Protocol tool definition)
- Server action: `generateApiDocsAction()` — editors/admins only
- Model: `claude-sonnet-4-5`

## 3. Semantic Search

- Natural language search across all APIs and projects
- Query by intent ("find endpoints that handle authentication")
- Claude API ranks results by semantic relevance
- Server action: `semanticSearchAction(query)`

## 4. API Import

### OpenAPI / Swagger Import
- Parses OpenAPI 3.x and Swagger 2.0 specs
- Extracts paths, methods, schemas, tags, summaries
- Bulk imports all extracted entries
- Component: `OpenAPIImportModal`

### HAR File Import
- Parses HTTP Archive files from browser DevTools
- Captures live API traffic (method, URL, request/response bodies)
- Infers JSON schemas from actual payloads
- Normalizes paths (UUIDs → `{id}`)
- Deduplicates similar endpoints, filters static assets
- Component: `HARImportModal`

## 5. Role-Based Access Control

Four roles with distinct permissions:

| Role | View | Suggest | Edit | Approve | Manage Users |
|------|------|---------|------|---------|--------------|
| Viewer | ✓ | — | — | — | — |
| Suggester | ✓ | ✓ | — | — | — |
| Editor | ✓ | ✓ | ✓ | ✓ | — |
| Admin | ✓ | ✓ | ✓ | ✓ | ✓ |

## 6. Suggestion & Approval Workflow

- Suggesters propose create / edit / delete changes (status: pending)
- Editors review field-by-field diffs via `DiffView` component
- Approve → change applied; Reject → optional reviewer note recorded
- Suggesters can withdraw their own pending suggestions
- Centralised `/suggestions` page with role-filtered views

## 7. Audit Trail & History

- Append-only `history_events` table per API entry
- Tracked actions: `created`, `edited`, `deleted`, `suggested_*`, `suggestion_approved`, `suggestion_rejected`, `bulk_import`
- `HistoryFeed` component shows timeline with actor, action, timestamp
- Visible in "History" tab on every API detail page

## 8. API Detail View (6-Tab Interface)

| Tab | Content |
|-----|---------|
| Overview | Endpoint, method, version, status, group, tags |
| Schema | Request & response JSON with syntax highlighting |
| MCP Config | Ready-to-use MCP tool JSON config |
| Code Snippet | Example usage code |
| Notes | Rate limits, edge cases, auth requirements |
| History | Full audit trail |

## 9. User Management (Admin)

- List all users with roles
- Change roles (viewer → suggester → editor → admin)
- Add users by email
- Delete users from the team
- Located at `/settings/users`

## 10. Authentication

- Email/password sign-in
- Google OAuth (Supabase)
- Auth callback route for OAuth flow

## 11. MCP Config Generation

- Each API entry has a complete MCP tool definition:
  - `name`, `description`, `inputSchema` (JSON Schema)
- Displayed in-app and copy-ready for direct use in MCP hosts
- Auto-generated by AI or manually authored

## What Makes This Different

1. **LLM-First** — `tool_description` field is curated for AI agent comprehension, not humans
2. **MCP-Native** — Generates ready-to-use MCP tool definitions without manual translation
3. **Semantic Search** — Intent-based API discovery, not just keyword matching
4. **AI-Assisted Drafting** — Claude drafts documentation; humans review and refine
5. **Structured Collaboration** — Suggestion + approval workflow with full diff review
6. **Multi-Source Ingestion** — Import from OpenAPI specs or live HAR network captures
7. **Complete Audit Trail** — Immutable history per API entry
